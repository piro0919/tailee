# CLAUDE.md

Tailee のリポジトリで Claude Code が作業する際のガイドラインです。

## Tailee とは

**ペットとの「なんでもない日常」が勝手に貯まり、勝手に日記がつき、アルバムになるサービス。**

犬がご飯を食べている姿は、意外と誰も撮らない。撮ろうと思わないから撮らない。そこを残す。
一日ぶんは何でもないが、一年ぶん積もると意味を持つ。ペットの夕飯はおよそ一万回で終わる、という有限性がこの製品の芯にある。

**「そこを残す」であって「そこだけを残す」ではない。** 見せられる絵となんでもない絵は、
普通のアルバムなら両方入っている。SNS 向けに撮りに行く動きが出ても問題ない
（2026-07-17 ユーザー「アルバムなんて、そこらへん両方入ってるのが普通じゃない？」）。
純度を守る方向に設計を寄せないこと。

### 決まっていること

| 項目           | 内容                                                                     |
| -------------- | ------------------------------------------------------------------------ |
| **主体**       | アルバム。見守りカメラではない                                           |
| **見る人**     | 基本は飼い主本人。ただし共有を否定しない（下2行）                        |
| **ライブ配信** | 見守りカメラの映像を、そのままライブ配信したい                           |
| **SNS**        | アプリ内に、共有のインスタのようなものを作りたい                         |
| **カメラ**     | 古いスマホを想定。ただし使うかはユーザー次第。専用ハードは作らない       |
| **置き方**     | 餌皿の横などに置きっぱなし。毎日同じ場所で起きる数分を狙う               |
| **家にいる時** | ウェイクワードで録画・録音を開始。餌皿を持っていて画面を触れないから声   |
| **手持ち撮影** | 画面のボタンで開始。突発的ないい場面はこちら。同じアルバムに入る         |
| **留守中**     | 見守りカメラとして動く。リアルタイム配信であって、録画も録音もしない     |
| **日記**       | 音声から自動生成。人間に実況させない                                      |
| **展開**       | 海外展開が前提。ひらがな・日本語依存の名前は不可                         |
| **立ち位置**   | 並ぶ棚はペットカメラ（Furbo等）ではなく、子供の成長アルバム（みてね等）  |

### 設計上の分かれ目（重要）

**日記は「実況させない」。** カメラに向かって「今ご飯を食べています」と説明させる設計にすると三日で終わる。
飼い主はもともとペットに話しかけている（「はい、ごはん」「昨日は残したのに」）。
**その独り言を拾って、機械の側が文章を立ち上げる。** 人間に余計な発話を一言もさせない。

映像から分かるのは「11時に日向で寝ていた」まで。
「今日ワクチンを打った」「はじめて他の犬と挨拶できた」は飼い主の頭の中にしかない。
声を**指示ではなく素材として**拾えば、そこも残る。

**ただし日記を製品の核として扱わないこと。** 2026-07-17 にユーザーは
「日記はそこまでなんだよな、あくまでアルバムが主体という感じ」と述べている。
日記はアルバムに付くものであって、日記のために作っているのではない。

## 名前

**サービス名は Tailee（テイリー）。** tail に由来する造語。2026-07-17 に決定。

検索した限り同名のブランド・アプリ・商標は無い。日本語の一般語ともぶつからない。
※ これは検索であって商標調査ではない。本気で押さえる段では別途必要。

### 未決

- **アルバムの単位名。** `Taileaf` が候補。leaf は一冊ではなく一枚を指すので、
  「一日ぶん＝一枚の Taileaf、積もって一冊」が意味的には正しい。
  ただし Tailee と Taileaf は /f/ 一音しか違わず、/f/ は最も落ちやすい音。
  声に出すと混同する（「Tailee でしたっけ、Taileaf でしたっけ」）。実画面に並べてから決める。
- **アルバムの提供の仕方。** まだ詰めきれていない。「年一とかかなー」という程度で、
  2026-07-17 時点で不明瞭。ここは決まっていないものとして扱う。
- **音声認識エンジン。** 決めていない。姉妹リポジトリが Vosk で通しているのは参照先であって、
  Tailee の採用決定ではない。なお Vosk は語彙ベースで、モデルの語彙にない語は認識しない。
  選ぶ場合、造語をウェイクワードにはできない。
- **ウェイクワード。** 後からカスタマイズできる想定で、名前とは分けて考える（Echo と Alexa の関係）。
  条件は「日常語でない」「同音語がない」「四拍以上」、加えて **Tailee と音が衝突しない**こと。
  ペットの名前（Bailey / ベイリー等）と似ると、犬を呼ぶたびに誤発火する。

## 未検証のリスク

技術的に未検証というだけのリスト。**出所は不明。**
元は「ユーザーが実装しながら確かめると判断して先送りした」「蒸し返して止めない」と書かれていたが、
2026-07-17 に確認したところ、少なくとも「最大の未知」という位置づけはユーザーの判断ではなかった。
先送りを本人が決めた証拠が無いので、決定として扱わない。

設計がこの上に乗る場面では、事実として扱わないこと。優先度も未定。

1. **置きっぱなし端末の待ち受けが持つか**
   常時マイクを開いて音声認識を回し続ける処理は、iOS Safari では画面を閉じると止まる可能性が高い。
   Android でも継続的な集音は削られる。ここが崩れるとウェイクワードという前提ごと崩れる。
   古い端末は「余るほど古く、かつ音声認識が回るほど新しい」を同時に満たす必要があり、窓は狭い。
2. **バッテリー。** 劣化した古い端末を充電しっぱなしで発熱させたまま何ヶ月も置く構成。膨らむ。人の家に置く物なので軽視しない。
3. **電源の本数。** 端末代はゼロでも、置く場所ごとにコンセントとケーブルが要る。

### 台数について

追加台数の原価がゼロなのは、ハードを売る競合（Furbo 等）が絶対に真似できない構造的な強み。
**複数台用意できるなら、それに越したことはない。** 台数が増えれば画角が生まれ、その分アルバムは良くなる。

最初に何台から始めるかは未決。
